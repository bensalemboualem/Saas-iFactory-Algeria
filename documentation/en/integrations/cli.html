CLI Integration

Use IAFactory Algeria AI directly from your terminal with our powerful command-line interface.

üíª Introduction

The IAFactory CLI brings AI capabilities to your terminal for scripting, automation, and quick access to AI models without leaving the command line.

üí° Features:
  • Interactive chat mode
  • Pipe input/output for automation
  • File processing and batch operations
  • Custom prompts and templates
  • Multi-model support
  • Shell integration

üì¶ Installation

Via npm

npm install -g @iafactory/cli

# Verify installation
iafactory --version

Via pip (Python)

pip install iafactory-cli

# Verify installation
iafactory --version

Binary Installation

# macOS/Linux
curl -sSL https://cli.iafactory-algeria.com/install.sh | bash

# Windows (PowerShell)
irm https://cli.iafactory-algeria.com/install.ps1 | iex

‚öôÔ∏è Configuration

Authentication

# Interactive login
iafactory auth login

# Or set API key
iafactory config set api-key YOUR_API_KEY

# Or via environment variable
export IAFACTORY_API_KEY="your_api_key"

Configuration File

Edit ~/.iafactory/config.yaml:

api:
  key: YOUR_API_KEY
  base_url: https://api.iafactory-algeria.com/v1

defaults:
  model: gpt-4.1
  temperature: 0.7
  max_tokens: 2000

prompts:
  system: "You are a helpful assistant."

output:
  format: text  # text, json, markdown
  color: true
  streaming: true

üöÄ Usage

Interactive Mode

# Start interactive chat
iafactory chat

# With specific model
iafactory chat --model gpt-4.1

# With custom system prompt
iafactory chat --system "You are a Python expert"

One-Shot Commands

# Simple query
iafactory "Explain quantum computing in simple terms"

# With model selection
iafactory --model claude-3.7-sonnet "Write a haiku about coding"

# JSON output
iafactory --format json "List 5 programming languages" | jq

Piping and Automation

# Process file content
cat README.md | iafactory "Summarize this document"

# Generate from template
echo "Username: john_doe" | iafactory "Generate a welcome email"

# Batch processing
find . -name "*.py" | while read file; do
  iafactory "Review this code" < "$file" > "${file}.review"
done

# Chain commands
curl https://api.example.com/data | \
  iafactory "Extract key insights" | \
  iafactory "Create executive summary"

üìã Commands

chat

Options:
  --model MODEL        AI model to use
  --temperature TEMP   Creativity (0-1)
  --system PROMPT      System prompt
  --history FILE       Load conversation history

generate

iafactory generate [OPTIONS] PROMPT

Options:
  --model MODEL        AI model to use
  --format FORMAT      Output format (text, json, markdown)
  --output FILE        Save to file
  --template NAME      Use saved template

file

# Process single file
iafactory file process INPUT.txt --prompt "Translate to French"

# Batch process directory
iafactory file batch ./docs/ --pattern "*.md" \
  --prompt "Add table of contents"

# Generate from file
iafactory file generate template.txt --variables vars.json

template

# List templates
iafactory template list

# Create template
iafactory template create code-review \
  --prompt "Review this code for:\n1. Bugs\n2. Performance\n3. Best practices"

# Use template
iafactory template use code-review < mycode.js

üîÑ Automation Examples

Automated Code Review

#!/bin/bash
# review-pr.sh

PR_FILES=$(git diff --name-only main...HEAD)

for file in $PR_FILES; do
  echo "Reviewing $file..."
  
  iafactory --model gpt-4.1 \
    --system "You are a senior code reviewer" \
    "Review this code and provide feedback" < "$file" \
    > "reviews/${file}.review"
done

# Aggregate reviews
cat reviews/*.review | iafactory \
  "Summarize these code reviews into a PR comment"

Log Analysis

#!/bin/bash
# analyze-logs.sh

# Tail logs and analyze errors
tail -f /var/log/app.log | \
  grep ERROR | \
  while read line; do
    echo "$line" | iafactory \
      --model gpt-4.1-mini \
      "Explain this error and suggest a fix" \
      >> error-analysis.txt
  done

Documentation Generation

#!/bin/bash
# generate-docs.sh

# Generate README from code

  --pattern "*.py" \
  --prompt "Generate API documentation in Markdown" \
  --output docs/API.md

  iafactory "Create a changelog grouped by: Features, Fixes, Improvements" \
  > CHANGELOG.md

üêö Shell Integration

Bash

# Add to ~/.bashrc
eval "$(iafactory completion bash)"

# Alias for quick access
alias ai='iafactory'
alias chat='iafactory chat'

# Function for quick code explanations
explain() {
  cat "$1" | iafactory "Explain this code"
}

Zsh

# Add to ~/.zshrc
eval "$(iafactory completion zsh)"

# Quick command helper
function ask() {
  iafactory "$*"
}

# Git commit message generator
function gcm() {
  git diff --cached | iafactory \
    --model gpt-4.1-mini \
    --format text \
    "Generate a conventional commit message" | \
    xargs git commit -m
}

PowerShell

# Add to $PROFILE
iafactory completion powershell | Out-String | Invoke-Expression

# Function for AI assistance
function Ask-AI {
    param([string]$Question)
    iafactory $Question
}

# Alias
Set-Alias ai Ask-AI

‚ú® Best Practices

1. Scripting

  • Use --format json for parseable output
  • Set --temperature 0.1 for consistent results
  • Cache responses with --cache flag
  • Handle API errors with proper exit codes

2. Performance

  • Use gpt-4.1-mini for simple tasks
  • Batch similar requests together
  • Use streaming for long responses
  • Set appropriate max_tokens limits

3. Security

  • Never commit API keys to version control
  • Use environment variables or secure config files
  • Sanitize input before passing to AI
  • Review generated code before execution
