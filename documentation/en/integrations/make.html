Make.com Integration

Integrate IAFactory Algeria AI into your Make.com scenarios for intelligent workflow automation.

üîÑ Introduction

Make.com (formerly Integromat) is a powerful visual automation platform. Connect it with IAFactory Algeria to add AI capabilities to your workflows: text processing, image generation, intelligent decision-making, and more.

üí° Use Cases:
  • Process emails with AI and auto-respond

  • Chatbot workflows

üì¶ Setup

1. Create Make Account

  1. Go to make.com
  2. Sign up for a free account
  3. Verify your email

2. Get IAFactory API Key

  1. Log in to app.iafactory-algeria.com
  2. Navigate to Settings ‚Üí API Keys
  3. Click Create New Key
  4. Copy your key (keep it secure!)

1. Intelligent Email Processing

Workflow:

  1. Gmail ‚Üí Watch Emails (trigger)

  3. Router (route based on AI classification)

Workflow:

  1. Airtable ‚Üí Watch Records
  2. HTTP ‚Üí IAFactory API (generate content)

  }],

  "max_tokens": 2000
}

Rate Limiting & Retry

  • Sleep module: Add delays between requests
  • Repeater: Retry failed requests
  • Error handler: Exponential backoff

‚ú® Best Practices

1. Optimization

  • Use gpt-4.1-mini for simple tasks (faster & cheaper)
  • Use gpt-4.1 for complex reasoning
  • Set appropriate max_tokens to avoid waste
  • Cache responses when possible

2. Security

  • Store API key in Make's Connection (not in scenario)
  • Use Data Store to encrypt sensitive data
  • Validate webhook signatures
  • Never log API keys in scenario history

3. Monitoring

  • Set up Error Notifications (Slack/Email)
  • Use Data Store to track usage
  • Monitor execution time and optimize
  • Regular scenario testing

üîß Troubleshooting

Error 401: Unauthorized

  • Check API key is correct
  • Verify Authorization header format: Bearer YOUR_KEY
  • Ensure key hasn't expired

Error 429: Rate Limit

  • Add Sleep modules between requests
  • Implement exponential backoff
  • Upgrade your IAFactory plan

Timeout Errors

  • Reduce max_tokens
  • Split large requests into smaller chunks
  • Increase timeout in HTTP module settings
