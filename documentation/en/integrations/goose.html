Goose is a command-line development assistant that can read your codebase, execute commands, and assist with development tasks. Connect it to IAFactory Algeria to use our models.

Via pip

pip install goose-ai

# Verify installation
goose --version

Via Homebrew (macOS)

brew install goose-ai

âš™ï¸ IAFactory Configuration

Configuration File

Create ~/.goose/config.yaml:

providers:
  iafactory:
    type: openai
    base_url: https://api.iafactory-algeria.com/v1
    api_key: ${IAFACTORY_API_KEY}
    models:
      - gpt-4.1
      - gpt-4.1-mini
      - claude-3.7-sonnet

default_provider: iafactory
default_model: gpt-4.1

settings:
  temperature: 0.7
  max_tokens: 4000
  stream: true

Environment Variables

# In ~/.bashrc or ~/.zshrc
export IAFACTORY_API_KEY="your_api_key"

# Or for a session
export IAFACTORY_API_KEY="your_api_key"
goose

ğŸ¨ Customization

Goosefile (Project Config)

Create .goosefile.yaml at project root:

name: MyProject
description: Node.js REST API

context:
  - README.md
  - docs/architecture.md
  - package.json

rules:
  - "Always write tests"
  - "Follow ESLint convention"
  - "Document public functions"
  - "Commits in English"

commands:
  test: npm test
  lint: npm run lint
  build: npm run build

ai_model: gpt-4.1
temperature: 0.3

Custom Prompts

Create ~/.goose/prompts/:

# prompts/code-review.yaml
name: code-review-expert
description: Expert code reviewer

system_prompt: |
  You are a senior developer expert in code review.
  
  For each file:
  1. Code quality (1-10)
  2. Strengths (3)
  3. Improvements (priorities)
  4. Concrete suggestions
  5. Security issues
  
  Format: Markdown with emojis

temperature: 0.2

Usage:

goose --prompt code-review-expert

âœ¨ Best Practices

1. Project Context

  • Create a good .goosefile.yaml
  • Keep README.md and docs updated
  • Use .gooseignore to exclude files

2. Collaboration

  • Share .goosefile.yaml with the team
  • Document custom prompts
  • Always review generated code

3. Security

  • Never commit config.yaml with API keys
  • Use environment variables
  • Add sensitive files to .gooseignore
