Rédiger des Prompts

Maîtrisez l'art de communiquer avec l'IA pour obtenir les meilleurs résultats.

Principes de Base

1. Clarté

Bon 
Écris une fonction Python qui calcule la moyenne d'une liste de nombres.
Gère les listes vides en retournant 0.
Mauvais 
fais un truc pour calculer

2. Contexte

Je suis développeur backend en Node.js.
Je travaille sur une API REST pour un site e-commerce.
J'ai besoin d'une fonction middleware pour valider les tokens JWT.

Contraintes :
- Utiliser jsonwebtoken v9
- Gérer l'expiration
- Logs des erreurs
- TypeScript

3. Spécificité

Vague Spécifique
"Aide-moi avec SQL" "Écris une requête SQL pour obtenir les 10 clients ayant dépensé le plus en 2024"
"Explique React" "Explique le concept de hooks en React avec un exemple de useState et useEffect"
"Débugue mon code" "Ce code Python retourne KeyError sur la ligne 15. Voici le code: [code]. Comment corriger?"

Formules de Prompts

Formule: Rôle + Tâche + Format

Tu es [RÔLE].

Tâche : [CE QUE TU VEUX]

Format de sortie : [COMMENT TU LE VEUX]
Exemple concret : 
Tu es un expert en cybersécurité.

Tâche : Analyse ce code PHP et identifie les vulnérabilités de sécurité.

Format de sortie :
- Liste numérotée des vulnérabilités
- Niveau de criticité (Faible/Moyen/Critique)
- Solution recommandée pour chacune

Formule: Contexte + Action + Contraintes

Contexte : [SITUATION]

Action : [CE QUI DOIT ÊTRE FAIT]

Contraintes :
- [CONTRAINTE 1]
- [CONTRAINTE 2]
- [CONTRAINTE 3]
Exemple : 
Contexte : Application mobile de fitness en React Native

Action : Crée un composant Timer pour les exercices

Contraintes :
- Utiliser React hooks
- Affichage format MM:SS
- Boutons play/pause/reset
- Notification sonore à la fin
- Style Material Design

Formule: Few-Shot Learning

Voici des exemples :

Exemple 1:
Input: [exemple 1 input]
Output: [exemple 1 output]

Exemple 2:
Input: [exemple 2 input]
Output: [exemple 2 output]

Maintenant fais la même chose pour:
Input: [ton input]
Exemple concret : 
Convertis ces phrases en format JSON structuré :

Exemple 1:
Input: "Jean Dupont habite à Paris, il a 30 ans"
Output: {"nom": "Jean Dupont", "ville": "Paris", "age": 30}

Exemple 2:
Input: "Marie Martin de Lyon, 25 ans"
Output: {"nom": "Marie Martin", "ville": "Lyon", "age": 25}

Maintenant convertis:
Input: "Ahmed Ben Ali vit à Alger, âgé de 35 ans"

Techniques Avancées

Chain-of-Thought (Chaînage de Pensée)

Résous ce problème étape par étape :

Problème : Calcule le coût total d'un projet

Données :
- 3 développeurs à 500€/jour
- 2 designers à 400€/jour
- 20 jours ouvrables
- +20% de charges

Réfléchis étape par étape avant de répondre.

Constraint-Based

Génère 5 noms de marque pour une startup de livraison de nourriture.

Contraintes OBLIGATOIRES :
- Maximum 8 lettres
- Facile à prononcer en français et arabe
- Disponible comme nom de domaine .com
- Évoque la rapidité ou la fraîcheur
- Pas de chiffres

Pour chaque nom, explique pourquoi il respecte les contraintes.

Iterative Refinement

Étape 1 : "Écris un article sur l'IA"
→ Réponse générique

Étape 2 : "Rends-le plus technique, cible : développeurs"
→ Ajout de détails techniques

Étape 3 : "Ajoute des exemples de code en Python"
→ Code ajouté

Étape 4 : "Simplifie pour les débutants en IA"
→ Article final optimisé

Anti-Patterns à Éviter

1. Trop Vague

"Aide-moi"
"Ça marche pas"
"Fais quelque chose avec ce code"

2. Trop Complexe en Une Fois

Mauvais : "Crée-moi une app complète avec frontend React, backend Node, base MongoDB, auth JWT, tests unitaires, CI/CD, Docker, et déploie sur AWS"

Bon : Décomposer en étapes :
1. "Structure backend Node.js avec Express"
2. "Ajoute système auth JWT"
3. etc...

3. Manque de Contexte

Mauvais : "Ce code a un bug"
[code sans contexte]

Bon : "Bug dans fonction de paiement Stripe.
Environnement : Node.js 18, stripe v13.
Erreur : [message exact]
Code : [snippet pertinent]"

4. Attentes Irréalistes

Mauvais : "Crée un AlphaGo en 10 lignes de Python"
Mauvais : "Génère une IA qui prédit la bourse à 100%"

Bon : Objectifs réalistes et décomposés

Évaluer la Qualité d'un Prompt

Checklist

  • Objectif clair ?
  • Contexte fourni ?
  • Contraintes spécifiées ?
  • Format de sortie défini ?
  • Exemples si nécessaire ?
  • Niveau de détail approprié ?
  • Vocabulaire précis ?
