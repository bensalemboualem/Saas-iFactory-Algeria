تقنيات الأوامر

1. كن محددًا

سيء: 
اكتب شيئًا عن Python
جيد: 
اكتب دالة Python تقوم بـ:
- أخذ قائمة من الأعداد الصحيحة
- إرجاع ثاني أكبر رقم
- التعامل مع الحالات الحدية (قائمة فارغة، تكرارات)
- تضمين docstring وتلميحات الأنواع

2. قدم السياق

أنا أبني تطبيق تجارة إلكترونية React مع TypeScript.
أستخدم Zustand لإدارة الحالة.
أحتاج مكون عربة تسوق يقوم بـ:
- عرض العناصر مع أدوات التحكم في الكمية
- حساب المجموع مع الضريبة (10%)
- الحفظ في localStorage

3. استخدم الأمثلة (Few-Shot)

حوّل هذه النصوص إلى JSON:

مثال 1:
المدخل: "أحمد علي، 25 سنة، يسكن في الجزائر"
المخرج: {"name": "أحمد علي", "age": 25, "city": "الجزائر"}

مثال 2:
المدخل: "فاطمة من وهران، عمرها 30"
المخرج: {"name": "فاطمة", "age": 30, "city": "وهران"}

الآن حوّل:
المدخل: "كريم، 28، من قسنطينة"

4. سلسلة التفكير

حل هذه المشكلة خطوة بخطوة:

المشكلة: احسب التكلفة الشهرية لفريق من 5 مطورين
- السعر الأساسي: 500$/مطور
- خصم 20% للفرق أكثر من 3
- 100$ إضافية للدعم المميز
- ضريبة 10%

فكر في كل خطوة قبل إعطاء الإجابة النهائية.

5. تعيين الدور

أنت مدقق أمني متمرس لديك 15 سنة من الخبرة.
راجع هذا الكود بحثًا عن الثغرات الأمنية:

[الصق الكود]

ركز على:
- حقن SQL
- ثغرات XSS
- عيوب المصادقة
- مشاكل تحديد المعدل

تحسين API

تقليل التكاليف

# استخدم نماذج mini للمهام البسيطة
response = client.chat.completions.create(
    model="gpt-4.1-mini",  # أرخص، لا يزال جيدًا
    messages=[...],
    max_tokens=500  # تحديد المخرجات
)

# تفعيل التخزين المؤقت للاستعلامات المتكررة
response = client.chat.completions.create(
    model="gpt-4.1",
    messages=[...],
    cache=True  # إعادة استخدام الردود المخزنة
)

تحسين السرعة

# استخدم البث للاستجابة الأسرع
for chunk in client.chat.completions.create(
    model="gpt-4.1-mini",
    messages=[...],
    stream=True
):
    print(chunk.choices[0].delta.content, end="")

# تجميع الطلبات عند الإمكان
responses = client.batch.create(
    requests=[
        {"model": "gpt-4.1-mini", "messages": [...]},
        {"model": "gpt-4.1-mini", "messages": [...]},
    ]
)

سيء: "ابنِ لي موقع تجارة إلكترونية كامل مع React، Node، MongoDB،

...
